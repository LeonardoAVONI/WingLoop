To explicitly record your agreement, add your name and the date to the
`CONTRIBUTORS.md` file as part of your pull request.

## Why a CLA?

Without a CLA, every contributor retains full copyright over their code.
This means that if WingLoop is ever relicensed (e.g. to offer a commercial
version), the author would need permission from every single contributor.
The CLA transfers the necessary rights upfront, keeping the project legally
manageable while still crediting all contributors.

This is standard practice for projects that maintain a dual-license model
(examples: Qt, MongoDB, MariaDB).

## How to Contribute

1. Fork the repository and create a branch for your changes.
2. Make your changes, following the existing code style.
3. Add your name to `CONTRIBUTORS.md` to confirm CLA acceptance.
4. Open a pull request with a clear description of what you changed and why.

For major changes, please open an issue first to discuss the proposal.
